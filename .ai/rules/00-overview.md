# 00 — Обзор и принципы

> Этот файл описывает базовые принципы работы AI-ассистентов в репозитории.
> Точка входа — корневой [`AGENTS.md`](../../AGENTS.md). Здесь — расшифровка принципов и навигация.

## Базовые принципы

1. **Read before write.** Перед изменением кода — прочитай релевантные файлы и правила. Перед изменением правил — спроси разрешения.
2. **Plan before act.** Для любой задачи сложнее тривиальной — сформулируй план (декомпозиция, файлы в скоупе, критерии приёмки). При работе через Cursor — план фиксируется в чате; при делегировании ptr_coder — в `.ai/handoffs/PTR-XXX-<slug>.md`.
3. **Verify before commit.** Перед коммитом — линт, форматер, типы, тесты. Перед PR — повторный полный прогон.
4. **Small, atomic, reversible.** Один PR = одна логическая задача. Размер PR — не более ~400 строк диффа без обоснования.
5. **Linear is the source of truth.** Каждое нетривиальное изменение должно быть привязано к задаче `PTR-XXX`.
6. **Human is the gatekeeper.** Агент *никогда* не мерджит свой PR. Финальное решение — за человеком.
7. **Auditability.** Всё, что делает агент, должно быть восстановимо по git-истории, Linear и комментариям PR. Не использовать недокументированные shell-обходы.
8. **Least privilege.** Если действие не описано в [правах автономии](90-forbidden.md), агент **спрашивает** перед его выполнением.

## Термины

| Термин | Значение |
|---|---|
| **Агент** | Любой AI-ассистент, исполняющий задачи в репозитории (Cursor, ptr_coder, …) |
| **Оркестратор** | Агент, который декомпозирует задачу и делегирует подзадачи другим агентам. По умолчанию — Cursor |
| **Кодер** | Агент, реализующий код по спецификации. По умолчанию — **ptr_coder** (LM Studio) |
| **Хэндофф** | Письменная спецификация задачи, передаваемая от оркестратора к кодеру. Файл `.ai/handoffs/PTR-XXX-<slug>.md` |
| **DoD** | Definition of Done — список условий, обязательных перед merge |
| **PR** | Pull Request на GitHub |

## Что делать, если правила противоречат друг другу

Приоритет (от высшего к низшему):

1. Явная инструкция пользователя в текущем диалоге.
2. [`90-forbidden.md`](90-forbidden.md) — запреты не обходятся.
3. [`AGENTS.md`](../../AGENTS.md) — корневой свод.
4. Файлы `.ai/rules/*.md` — частные правила.
5. Решения в `docs/adr/` — обоснования архитектурных выборов.

При обнаружении противоречия — **остановись и сообщи пользователю**, не пытайся разрешить молча.
