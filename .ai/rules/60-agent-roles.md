# 60 — Роли агентов

> Краткая версия в [`AGENTS.md` §3](../../AGENTS.md#3-роли-агентов). Здесь — детальная спецификация.

## Cursor — оркестратор / архитектор / валидатор

**Когда работает:** в IDE Cursor (чат + Background Agents).

**Ответственность:**

1. **Декомпозиция задач.** Берёт задачу из Linear, разбивает на шаги, при необходимости создаёт под-задачи.
2. **Архитектура.** Принимает архитектурные решения, оформляет их в ADR (`docs/adr/NNNN-*.md`).
3. **Спецификация для ptr_coder.** Готовит хэндофф-документ `.ai/handoffs/PTR-XXX-<slug>.md` с целью, файлами в скоупе, критериями приёмки и тест-планом.
4. **Делегирование.** Запускает **ptr_coder** (`python -m ptr_coder`) как subprocess с подготовленной спецификацией (см. [`70-orchestration-ptr-coder.md`](70-orchestration-ptr-coder.md)).
5. **Валидация результата.** Прогоняет тесты, линтеры, типы; визуально/функционально проверяет реализацию против критериев приёмки; при необходимости — итерирует с уточнённой спекой.
6. **Оформление PR.** Финальный коммит-полишинг, заголовок PR, заполнение тела, связь с Linear.

**Что Cursor делает сам, без делегирования:**

- Документация (`AGENTS.md`, `.ai/rules/`, ADR, README).
- Конфиги CI и инфраструктуры монорепы (но строго **с явного запроса пользователя** — это в чёрном списке для общей автономии, см. [`90-forbidden.md`](90-forbidden.md)).
- Лёгкий рефакторинг и правки, где спецификация = «прочитать и поправить очевидно».

## ptr_coder — кодер (LM Studio)

**Когда работает:** запускается Cursor как subprocess (`python -m ptr_coder`, см. [`70-orchestration-ptr-coder.md`](70-orchestration-ptr-coder.md)) или человеком вручную для отладки.

**Персоны.** Оркестратор выбирает персону (`backend`, `frontend`, `designer`, `qa`, `infra`, `docs`) и фиксирует её в секции `## Persona` хэндоффа — как контекст для модели и для Cursor при валидации. Подробности — [`65-personas.md`](65-personas.md).

**Ответственность (модель + инструменты ptr_coder):**

1. Реализовать код **строго по хэндофф-спецификации** в рамках разрешённых tools (`read_file`, `write_file`, `list_directory`) и корня `--root`.
2. По возможности следовать командам из секции `## Commands` в handoff (фактический прогон и интерпретацию результатов выполняет Cursor при валидации).
3. Не коммитит и не пушит сам по себе: изменения остаются в рабочем дереве; коммиты делает Cursor (или человек) после проверки.

**Чего ptr_coder НЕ делает:**

- Не выходит за пределы `## Files in scope` / политики путей; вне скоупа — только сообщение в траектории диалога, **без** правок файлов.
- Не редактирует `.ai/handoffs/*.md` (включая `## Result` — это Cursor).
- Не открывает PR и не трогает Linear.
- Не заменяет архитектурные решения; при неоднозначности спеки — текстом запрашивает уточнение у оркестратора.

## Codex CLI cloud — генератор изображений

**Когда работает:** вызывается, когда задача требует графических ассетов (иконки, иллюстрации, скриншоты для документации, моки UI). Типичный контекст — бриф персоны **designer** в [`65-personas.md`](65-personas.md#designer): оркестратор передаёт описание ассета и целевой каталог, затем забирает файлы в PR.

**Ответственность:**

- Сгенерировать ассет по описанию.
- Сохранить в каталог, согласованный со стеком (для документации — `docs/images/`, для веба — `apps/<app>/public/` и т. п.; конкретные пути — по мере роста монорепы).
- Передать управление обратно вызывающему агенту.

**Не используется для:** написания кода, конфигов, документации.

## Человек (`@enotbert`) — финальный ревьюер

**Ответственность:**

1. Постановка задач в Linear (или верификация задач, созданных агентами).
2. Code review всех PR.
3. Merge PR (squash-merge).
4. Решения по чёрному списку: изменение зависимостей, CI, правил агентов, миграции БД, релизы.
5. Конфликтное разрешение, когда правила противоречат друг другу или агент остановился из-за неоднозначности.

## Как агенты сообщают друг другу о результатах

| Из | В | Канал |
|---|---|---|
| Cursor | ptr_coder | Хэндофф `.ai/handoffs/PTR-XXX-<slug>.md` + `python -m ptr_coder --handoff … --root …` |
| ptr_coder | Cursor | stdout/stderr процесса + изменения файлов в рабочем дереве; раздел `## Result` в handoff заполняет **Cursor** после прогона |
| Cursor (после брифа designer) | Codex CLI cloud | Текстовое описание ассета + целевой путь в репо; сгенерированные файлы попадают в рабочее дерево для того же PR или follow-up коммита |
| Любой агент | Человек | Комментарий в Linear / тело PR / сообщение в чате Cursor |
| Человек | Агент | Описание задачи в Linear / комментарий в PR / прямое сообщение в чате Cursor |

## Замена ролей и вход новых инструментов

Любое расширение состава агентов (Claude Code, GitHub Copilot, Aider, новые модели Codex) или смена ответственностей оформляется через **ADR** в `docs/adr/`. Без ADR агент не может «передать» свои обязанности другому инструменту.
